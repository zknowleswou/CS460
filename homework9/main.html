More in Angular6

For this post I am going to cover working with Angular6. Specifically I am going to look at routing and components. Routing is an essential part of an SPA. Angular6 has a powerful routing engine that even includes things like lazy loading feature modules. Components are the building blocks of an SPA. Components are supposed to be designed in such a way that they are extensible and reusable throughout the SPA.

Defining routes and their respective components.

The basic flow is as follows. A route is defined like as in the screen shot above. Each route has a component registered to it. Each component has a template that it renders when called. That component can then render sub components to build features.

Component hooks

Components come with all sorts of hooks that allow for bi directional data sharing and event pushing. I used many of these hooks when creating the workout designer feature of my app.

The workout designer.

Partial code for the workout designer.

Circled in red on both images is a component and how the component is rendered by its parent component. In this case the parent component is the builder component and the child component is the dayListBuild component. There are two different types of hooks on the component.

The first hook is [allDays]="workout.days". That is a data binding that passes in the allDays collection into the component. The image below shows how the component consumes the collection being passed in. The second hook is (daySet)="setActiveDay($event)". This hook is an event that is published by the dayListBuild component. This particular event is making bubbling up a onClick event in the dayListBuild component.

Putting it all together

There is a lot more to Angular6. However, what I've shown are the most basic building blocks for any angular SPA. If I had more time I would have touched on the Angular6 bootstrap module that creates native Angular6 components to replace all bootstrap4 jQuery. Or how components can be created using *ngFor to create extremely powerful lists. Examples of those things can be found in my source code which is in the assignment code folder of homework 7.

My recommendation for teaching a framework would be Vue.Js. It is a very simple framework and works great with jQuery. I would suggest using jQuery to handle all of the AJAX requests and pass the data on to Vue.Js and let it handle the presentation. Angular6 is too heavy to teach in a single term. I am by no means an expert on the topic but I hope I was able to share some interesting information. Thank you for reading.
